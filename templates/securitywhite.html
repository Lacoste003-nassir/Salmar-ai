Executive Summary

Salmar is an AI platform built on the premise that trust is earned through verifiable security. We engineer for confidentiality, integrity, and availability—augmented by transparency and user control. Our layered defense model extends from cryptography and infrastructure to AI safety, incident response, and compliance.

“Salmar is designed with privacy, integrity, and resilience at its core. Security is not a feature; it is the foundation.”

Highlights

  • End-to-end encryption for data in transit and at rest.
  • Tamper-resistant, verifiable logs for auditability.
  • Redundant architecture with measurable SLOs and disaster recovery.
  • User-centric controls: consent, transparency, and data minimization.

Security Principles

Confidentiality

We employ strong cryptography and access controls so data remains protected end-to-end. Access is restricted on a least-privilege basis.

Integrity

Critical operations are logged immutably with integrity checks and cryptographic signing to deter tampering and support forensics.

Availability

Salmar’s services are designed for resilience: multi-region redundancy, autoscaling, and robust DDoS defenses ensure continuity.

User Control

Privacy by default: transparent processing, clear consent, and self-service data controls.

3

Data Protection & Privacy

Encryption Standards

  • Data in transit: TLS 1.3 with modern cipher suites and forward secrecy.
  • Data at rest: AES-256 with envelope encryption and key rotation.
  • Key management: Segregated KMS, HSM-backed keys where applicable.

Privacy-First Design

  • Minimal data collection aligned to purpose limitation.
  • Structured anonymization and pseudonymization of datasets.
  • Data retention with deletion SLAs and regional residency options.
Biometric Authentication
Stored only on device. Never uploaded to Salmar servers. Platform-native secure enclaves (e.g., Secure Enclave, TPM) enforce isolation. Salmar receives only proof-of-presence or platform attestation—never raw biometric data.
4

Authentication & Access Control

Authentication

  • Multi-factor authentication (MFA): TOTP, WebAuthn security keys, enterprise SSO.
  • Biometric login: Fingerprint, Face ID, Windows Hello via platform authenticators.

Authorization

  • Role-based access (RBAC) with hierarchical scopes and service accounts.
  • Just-in-time elevation with approvals and time-bound policies.

Session Security

  • Short-lived tokens with audience binding and rotation.
  • Automatic session expiration and revocation on risk signals.

Infrastructure Security

  • Cloud provider certifications: ISO 27001, SOC 2, GDPR-aligned processing.
  • Network segmentation, private subnets, WAF and layer-7 firewalls.
  • DDoS protection and anomaly detection with automated mitigation.
  • Continuous monitoring (SIEM), runtime security, and automated response.

Appendices

A.1 Glossary

  • Forward Secrecy: A property of secure communication protocols where compromise of long-term keys does not compromise past session keys.
  • WebAuthn: A W3C standard enabling strong, phishing-resistant authentication using public-key cryptography.
  • RBAC: Role-Based Access Control, assigning permissions to roles rather than individuals.

A.2 Data Retention & Deletion

  • Default retention minimized; customer-configurable retention windows.
  • Cryptographic erasure for keys protecting data scheduled for deletion.
  • Deletion SLAs documented in enterprise agreements.
